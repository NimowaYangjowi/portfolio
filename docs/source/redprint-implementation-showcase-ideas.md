# Redprint Portfolio Implementation Ideas

This folder contains Korean and English versions of the Redprint portfolio implementation outline.

이 파일은 안내판이고 실제 본문은 아래 두 파일이다.

## Versions

- Korean: `redprint-implementation-showcase-ideas.ko.md`
- English: `redprint-implementation-showcase-ideas.en.md`
- Korean admin/worker coverage: `redprint-admin-worker-coverage.ko.md`
- English admin/worker coverage: `redprint-admin-worker-coverage.en.md`
- Source trace: `redprint-implementation-source-trace.md`

## Language Direction

Both versions use the same framing:

- Avoid titles that are too generic, such as `Prompt Form` or `Payment Page`.
- Avoid titles that are too internal, such as `creator-facing publishing submission coordinator`.
- Use product-context titles, then explain the technical implementation details below.

Example:

- Korean: `복구 가능한 콘텐츠 등록 흐름`
- English: `Recoverable Content Publishing Flow`
- Technical details: `multi-step workflow`, `draft persistence`, `media readiness gates`, `submit coordinator`
